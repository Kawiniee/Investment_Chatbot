# AI Investment Advisor Chatbot
เอเจนต์สนทนาภาษาไทยที่ให้คำแนะนำการลงทุน, การคำนวณ, และคำแนะนำพอร์ตการลงทุนโดยใช้ LangChain, LangGraph, โมเดล Google Gemini, และ ChromaDB สำหรับการค้นหาเวกเตอร์ อินเทอร์เฟซสร้างด้วย Gradio และทำงานในเครื่อง
## คุณสมบัติ
- **ฐานความรู้การลงทุน** – เอกสารที่โหลดล่วงหน้าเกี่ยวกับหุ้น, กองทุน, พันธบัตร, ทอง, และกลยุทธ์
- **เครื่องมือ**
  - `calculate_investment_return` – การคำนวณดอกเบี้ยทบต้น
  - `search_investment_info` – การค้นหาเชิงความหมายในฐานความรู้
  - `recommend_portfolio` – คำแนะนำพอร์ตการลงทุนตามระดับความเสี่ยง (ต่ำ/ปานกลาง/สูง)
  - `calculate_dca` – มูลค่าในอนาคตของการลงทุนรายเดือนปกติ (Dollar Cost Averaging)
- **เอเจนต์** – เอเจนต์ LangChain ที่ตัดสินใจว่าจะใช้เครื่องมือใด, ถูกชี้นำโดยพรอมต์ระบบภาษาไทย
- **การบันทึก** – การดำเนินการทั้งหมดถูกบันทึกไปยัง `logs/agent_YYYYMMDD.log` และพิมพ์ไปยังคอนโซล
- **อินเทอร์เฟซเว็บ** – อินเทอร์เฟซแชทง่ายๆ ผ่าน Gradio (http://localhost:7860)
## การติดตั้ง
1. โคลนที่เก็บและนำทางไปยังโฟลเดอร์โปรเจกต์
2. สร้างสภาพแวดล้อมเสมือน (ไม่บังคับแต่แนะนำ):
   ```bash
   python -m venv venv
   source venv/bin/activate   # Windows: venv\Scripts\activate
3. ติดตั้งการพึ่งพา:
      pip install -r requirements.txt
4. ตั้งคีย์ API ของ Google ในไฟล์ .env (คัดลอกจาก .env.example ถ้ามี):
      GOOGLE_API_KEY=your_actual_api_key_here

วิธีการทำงาน
1. สภาพแวดล้อม & การบันทึก – โหลดคีย์ API, สร้างไดเรกทอรีบันทึก, และกำหนดค่าการบันทึก
2. โมเดล – เริ่มต้นโมเดลแชท Gemini (gemini-2.5-flash) และโมเดล embedding (gemini-embedding-001)
3. Vector Store – โหลดเอกสารการลงทุนขนาดเล็กเข้า ChromaDB สำหรับการค้นหาความคล้ายคลึง
4. เครื่องมือ – สี่ฟังก์ชันที่ตกแต่งด้วย @tool ที่ทำการคำนวณหรือค้นหา; แต่ละฟังก์ชันบันทึกอินพุต/เอาต์พุต
5. เอเจนต์ – รวม LLM, เครื่องมือ, และพรอมต์ระบบภาษาไทยเข้าเอเจนต์ LangChain ด้วย checkpointing ในหน่วยความจำ
6. อินเทอร์เฟซ Gradio – ห่อเอเจนต์ในฟังก์ชันแชทที่สร้าง thread ID เฉพาะต่อการสนทนา, เรียกเอเจนต์, และส่งคืนการตอบสนองของ AI
7. หลัก – พิมพ์ข้อมูลเริ่มต้นและเปิดเซิร์ฟเวอร์ Gradio
การขยายบอท
- เพิ่มความรู้เพิ่มเติม – วางไฟล์ .txt หรือ .json เพิ่มเติมในโฟลเดอร์ data/ และโหลดเข้า investment_docs
- กำหนดค่าเกณฑ์ – ย้ายการตัดคะแนนความคล้ายคลึง (0.8) และชื่อโมเดลไปยังตัวแปรสภาพแวดล้อมหรือไฟล์ config.py
- บันทึก Checkpoint – แทนที่ InMemorySaver ด้วย saver ที่คงอยู่ (เช่น SQLite) เพื่อเก็บประวัติการสนทนาไว้ข้ามการรีสตาร์ท
- ปรับใช้ – ใช้ Docker หรือบริการคลาวด์ (เช่น Hugging Face Spaces) เพื่อโฮสต์แอป Gradio
